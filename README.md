### 🤖 &nbsp; SentientML &nbsp;—&nbsp; AutoML Platform

<p>
  <a href="https://sentientml.site"><img src="https://img.shields.io/badge/Live_Demo-22c55e?style=flat-square&logo=vercel&logoColor=white"/></a>
  &nbsp;
  <a href="https://github.com/rajaharisai/sentientml"><img src="https://img.shields.io/badge/Architecture_Docs-181717?style=flat-square&logo=github&logoColor=white"/></a>
  &nbsp;
  <a href="mailto:hello@rajaharis.com"><img src="https://img.shields.io/badge/Request_Code_Access-EA4335?style=flat-square&logo=gmail&logoColor=white"/></a>
</p>

Upload a raw dataset. Receive a deployed, production-ready predictive model. No configuration required.

The system runs a **5-algorithm tournament** to screen candidates, selects a single champion, then applies **Bayesian hyperparameter optimization via Optuna TPE** with adaptive compute allocation across six effort tiers. Preprocessing decisions — scaler selection, imputation strategy, SMOTE gating, feature engineering — are made autonomously based on the statistical profile of the incoming data. The final export is a self-contained FastAPI server, CLI script, and Dockerfile.

### 📄 &nbsp; AskMyDocs &nbsp;—&nbsp; Hybrid RAG Document Engine

<p>
  <a href="https://askmydocs-ten.vercel.app"><img src="https://img.shields.io/badge/Live_Demo-22c55e?style=flat-square&logo=vercel&logoColor=white"/></a>
  &nbsp;
  <a href="https://github.com/rajaharisai/askmydocs"><img src="https://img.shields.io/badge/Architecture_Docs-181717?style=flat-square&logo=github&logoColor=white"/></a>
  &nbsp;
  <a href="mailto:hello@rajaharis.com"><img src="https://img.shields.io/badge/Request_Code_Access-EA4335?style=flat-square&logo=gmail&logoColor=white"/></a>
</p>

A hallucination-resistant document QA platform built on a three-layer retrieval stack: **vector search** (ChromaDB + MiniLM) for semantic intent, **BM25** for exact keyword precision, and a **cross-encoder reranking pass** to resolve the bi-encoder approximation gap. Handles scanned PDFs and handwritten documents via EasyOCR fallback. Answers stream token-by-token via SSE with granular page-level citations. A hard-fail grounding instruction prevents generation when retrieved context is insufficient.

### 📝 &nbsp; REZUMAI &nbsp;—&nbsp; Agentic Resume Optimizer

<p>
  <a href="https://rezumai-amber.vercel.app"><img src="https://img.shields.io/badge/Live_Demo-22c55e?style=flat-square&logo=vercel&logoColor=white"/></a>
  &nbsp;
  <a href="https://github.com/rajaharisai/rezumai"><img src="https://img.shields.io/badge/Architecture_Docs-181717?style=flat-square&logo=github&logoColor=white"/></a>
  &nbsp;
  <a href="mailto:hello@rajaharis.com"><img src="https://img.shields.io/badge/Request_Code_Access-EA4335?style=flat-square&logo=gmail&logoColor=white"/></a>
</p>

A stateless agentic system that identifies skill gaps between a resume and job description using a **multi-pass self-reflection loop** — the agent critiques its own analysis for shallow reasoning and overconfident scores before finalizing. Scoring combines 70% LLM semantic reasoning (Llama 3.3 70B via Groq) with 30% TF-based cosine similarity for technical grounding. Every internal reasoning step — chunking, retrieval, self-critique, refinement — streams live via NDJSON.

### 🏗️ &nbsp; Orchestrion &nbsp;—&nbsp; Enterprise AI Orchestration Architecture

<p>
  <a href="https://github.com/rajaharisai/orchestrion"><img src="https://img.shields.io/badge/Architecture_Repository-181717?style=flat-square&logo=github&logoColor=white"/></a>
  &nbsp;
  <img src="https://img.shields.io/badge/Status-Research_Sandbox-02569B?style=flat-square"/>
</p>

An ambitious, two-month architectural exploration into building a massive-scale, multi-agent orchestration platform. By heavily leveraging AI code generation to rapidly prototype 15+ subsystems across a Python DDD backend, a Next.js frontend, and PostgreSQL, I intentionally pushed the limits of what AI tools can independently build. The project served as a stress test, revealing the exact friction points where AI-generated system boundaries begin to break down without rigorous automated testing.

Rather than forcing the decoupled subsystems into a monolithic application, I open-sourced the repository as a high-value research testbed. It houses complete, reviewable implementations of **LangGraph** stateful execution, a custom **Policy Engine** (rate & cost caps), **Security Guardrails** (PII/prompt injection), a Fernet-encrypted secrets vault, and 30+ zero-auth tools. Deconstructing these immense integration challenges gave me the exact system-design maturity required to rapidly scope and ship the true production-grade products listed above.
